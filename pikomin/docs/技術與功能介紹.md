# Pikomin 技術與功能介紹

## 專案在做什麼

Pikomin 是一個 iOS GPS 模擬工具，主要用途是：

- 在地圖上設定定位點
- 規劃一段路線（多個路徑點）
- 讓手機沿著這段路徑自動移動（種花模式）

它適合用在需要測試定位功能的情境，例如：

- 地圖 App 測試
- 到點觸發流程測試
- 路線移動邏輯驗證

---

## 系統架構（簡單版）

整體由三部分組成：

1. 前端（Web UI）
2. 後端 API（控制與路徑計算）
3. iPhone 連線通道（tunneld + pymobiledevice3）

流程大致是：

1. 使用者在前端點地圖加入路徑點
2. 前端呼叫後端 `/api/route/start`
3. 後端路徑引擎計算每一步位置
4. 後端透過 `pymobiledevice3` 送定位到 iPhone
5. 前端透過 WebSocket / 狀態更新顯示藍點進度

---

## 使用技術

### 前端

- React + TypeScript
- Vite
- Leaflet（地圖顯示與路徑繪製）
- WebSocket（即時狀態）

### 後端

- Python
- FastAPI
- Uvicorn
- asyncio（路徑迴圈、非同步任務）

### 裝置連線與定位

- `pymobiledevice3`
- `tunneld`（iOS 17+ / 18+ 的連線方式）
- TCP tunnel（iOS 18.2+ 建議）

---

## 核心功能

### 1) 裝置偵測與選擇

- 顯示目前可用 iPhone
- 選擇控制目標裝置

### 2) 單點定位

- 點地圖後可直接把手機定位移到該點

### 3) 路徑模式（種花模式）

- 在地圖上連續加入多個路徑點
- 一鍵開始自動移動
- 可暫停 / 繼續 / 停止
- 可設定速度
- 可循環播放路線

### 4) 路徑資料管理

- 右側面板顯示路徑點列表
- 支援單筆移除與清除全部

### 5) 即時狀態回饋

- 顯示目前進度（百分比）
- 顯示目前座標位置
- 連線異常時可提示錯誤

---

## 目前運作特性（重要）

- 種花模式是「電腦主控」：
  - 路徑計算與座標推送在電腦端後端執行
  - 手機端被動接收定位
- 因此執行中需要：
  - 後端持續在線
  - tunnel 連線穩定
  - 手機保持可連線狀態

---

## 啟動需求（建議）

- Python 3.13（給 iOS 18.2+ TCP tunnel）
- `pymobiledevice3`
- Node.js + pnpm/npm（前端）

建議啟動方式：

1. 先啟 `tunneld`（TCP）
2. 再啟後端與前端（`./start.sh`）

---

## 常見問題

### 問：畫面在跑，但手機不動？

通常是 tunnel 或裝置連線層問題，不是 UI 問題。  
可先檢查 `http://127.0.0.1:49151` 是否有有效 tunnel 資料。

### 問：藍點不順或有拉扯？

通常是即時狀態流（WebSocket）延遲或不同更新來源覆蓋造成，需看前端同步策略。
