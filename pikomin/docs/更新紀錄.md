# 更新紀錄

## 2026-05-30

本次補充更新新增明信片瀏覽圖層，並整理右鍵資訊卡、類型篩選與 Pikmin Atlas API 代理。

### 明信片圖層

- 新增右側工具列明信片開關，開啟後會收起左側操作面板，讓地圖保留給明信片瀏覽。
- 明信片資料改由後端代理 `https://pikmin-atlas.com/api/postcards/nearby`，前端只渲染目前地圖視窗範圍內的項目。
- 後端新增 `POST /api/postcards/nearby`，標準化明信片名稱、座標、圖片、標籤與持有人數，並加入 90 秒短快取。
- 明信片以固定卡片樣式顯示圖片與名稱，避免橫圖、直圖造成 marker 跑版。

### 明信片篩選與操作

- 畫面上方新增明信片類型篩選列，支援全部、廟宇、變電箱、教堂與公園。
- 在全部狀態下點單一類型會切換為只顯示該類；非全部狀態下可複選多個類型。
- 右鍵明信片會顯示資訊卡，包含名稱、座標、複製資訊與儲存為地標圖示按鈕。
- 儲存地標預設新增為花點，方便後續從地標功能快速飛到該位置。

### 驗證

- 前端型別檢查：`tsc --noEmit`，通過。
- 後端 Python 編譯檢查：`py_compile`，通過。
- 本地 `POST /api/postcards/nearby` 代理測試，通過。

本次更新聚焦在種花模式的定位穩定性，降低開始路線時被拉回、路線停止後畫面回到舊座標，以及 host bridge 推送延遲造成的問題。

### 種花與路線穩定性

- 種花速度固定回 `15 km/h`，避免過快速度造成座標跳點太大。
- 路線啟動時會從目前最後定位座標銜接到第一個路徑點，不再一開始直接跳回第一個點。
- 路線停止或自然完成後，後端保留最後一次成功推送的座標，避免前端輪詢狀態時顯示回舊位置。
- 單點定位與重設 GPS 後會同步更新後端路線狀態，並透過 WebSocket 推送給前端。

### Host Bridge

- Host bridge 改為優先使用持久 `LocationSimulation` 連線設定 GPS，減少每次定位都重新啟動 CLI 的延遲。
- 新增 Windows 開發用 `scripts/start_windows_dev_bridge.ps1`，可單獨啟動 host bridge 與 tunneld。

### 介面

- 路線工具列改為圖示化操作，支援儲存、匯入、讀取與清除路徑。
- 路徑點支援右鍵操作，可顯示座標、設為起點、設為終點或移除節點。
- 未偵測到裝置時新增說明按鈕，方便查看連線提示。

### 驗證

- 前端型別檢查：`tsc --noEmit`，通過。
- 後端 route engine smoke check，通過。

## 2026-05-29

本次更新聚焦在地標管理、路徑操作、裝置辨識與啟動穩定性，讓桌面使用流程更清楚、可編輯，也更容易確認目前操作的裝置。

### 地標管理

- 新增地標編輯功能，可修改已儲存地標的名稱、經緯度與類型（花點 / 菇點）。
- 地標管理彈窗改為桌面雙欄配置：左側新增 / 編輯表單，右側顯示已儲存地標清單。
- 右側清單支援搜尋與類型篩選，並提供「編輯」與「刪除」操作。
- 點選「編輯」後，地標資料會帶入左側表單，按鈕切換為「更新」，並可取消編輯。
- 座標格式無效時，會顯示錯誤提示並清空座標欄位，方便重新輸入。
- 後端新增 `PUT /api/landmarks/{landmark_id}`，用於更新地標資料。

### 路徑操作

- 路徑點支援拖曳調整，像錨點一樣可直接在地圖上改位置。
- 拖曳路徑點後，路徑線會依新位置重畫。
- 拖曳時會避免誤觸地圖點擊，防止多新增路徑點。
- 移動方式新增 `跑步 (15 km/h)` 選項。

### 裝置與連線

- 裝置清單改為只顯示 USB 有線連接中的 iPhone。
- 裝置下拉改顯示手機名稱與型號，避免只看到 UDID 難以辨識。
- Host bridge 新增 USB 裝置查詢與裝置資訊查詢能力，供後端取得有線裝置與手機名稱。
- Docker stack 啟動流程調整為使用固定 host bridge 管理方式，降低 bridge 沒有常駐造成定位橋接失敗的機率。

### 介面與版本

- 主面板「裝置與狀態」旁新增程式版本號顯示，目前版本來源為 `frontend/package.json`。
- 前端目前顯示版本為 `v0.1.0`。

### 驗證

- 後端測試：`pytest`，30 passed。
- 前端測試：`npm run test`，9 passed。
- 前端建置：`npm run build`，通過。
