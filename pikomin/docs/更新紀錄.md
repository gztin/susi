# 更新紀錄

## 2026-05-29

本次更新聚焦在地標管理、路徑操作、裝置辨識與啟動穩定性，讓桌面使用流程更清楚、可編輯，也更容易確認目前操作的裝置。

### 地標管理

- 新增地標編輯功能，可修改已儲存地標的名稱、經緯度與類型（花點 / 菇點）。
- 地標管理彈窗改為桌面雙欄配置：左側新增 / 編輯表單，右側顯示已儲存地標清單。
- 右側清單支援搜尋與類型篩選，並提供「編輯」與「刪除」操作。
- 點選「編輯」後，地標資料會帶入左側表單，按鈕切換為「更新」，並可取消編輯。
- 座標格式無效時，會顯示錯誤提示並清空座標欄位，方便重新輸入。
- 後端新增 `PUT /api/landmarks/{landmark_id}`，用於更新地標資料。

### 路徑操作

- 路徑點支援拖曳調整，像錨點一樣可直接在地圖上改位置。
- 拖曳路徑點後，路徑線會依新位置重畫。
- 拖曳時會避免誤觸地圖點擊，防止多新增路徑點。
- 移動方式新增 `跑步 (15 km/h)` 選項。

### 裝置與連線

- 裝置清單改為只顯示 USB 有線連接中的 iPhone。
- 裝置下拉改顯示手機名稱與型號，避免只看到 UDID 難以辨識。
- Host bridge 新增 USB 裝置查詢與裝置資訊查詢能力，供後端取得有線裝置與手機名稱。
- Docker stack 啟動流程調整為使用固定 host bridge 管理方式，降低 bridge 沒有常駐造成定位橋接失敗的機率。

### 介面與版本

- 主面板「裝置與狀態」旁新增程式版本號顯示，目前版本來源為 `frontend/package.json`。
- 前端目前顯示版本為 `v0.1.0`。
