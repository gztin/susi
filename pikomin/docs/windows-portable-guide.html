Portable guide

不用安裝 Python 或 Node.js，解壓後即可在 Windows 本地端啟動 Pikomin。

這份文件給使用者或測試者閱讀。照順序完成準備、啟動、停止與排錯，就能用瀏覽器控制已連線 iPhone 的模擬定位。

執行入口 雙擊 run.bat
停止入口 雙擊 stop.bat
本機網址 http://localhost:5679

使用前準備

需要

Windows 10/11、Apple Mobile Device Support、已信任此電腦的 iPhone、USB 連線或可用的裝置連線。

不需要

不需要另外安裝 Python、Node.js、npm、FastAPI 或 pymobiledevice3。可攜版已內建 runtime。

項目 說明
Apple 驅動 Windows 必須能辨識 iPhone。可安裝 iTunes 或 Apple Devices 相關元件。
手機狀態 保持 iPhone 解鎖，第一次連線時在手機上點選「信任此電腦」。
系統權限 run.bat 會跳出系統管理員權限提示，用來啟動 iOS tunnel。
網路 地圖底圖需要連線讀取圖磚。沒有網路時功能仍可能可用，但地圖底圖會不完整。

啟動與停止

  1. 1 解壓縮 pikomin-win-portable.zip 到一個固定資料夾，例如 Documents\pikmin。
  2. 2 雙擊 run.bat。Windows 跳出權限提示時選擇「是」。
  3. 3 瀏覽器會自動開啟 http://localhost:5679。如果尚未開啟，可手動輸入網址。
  4. 4 使用完成後雙擊 stop.bat，停止背景中的 tunnel 與 app server。
解壓縮 run.bat 允許 UAC 瀏覽器開啟 stop.bat 停止

基本操作

功能 操作方式 注意事項
選擇裝置 開啟頁面後確認裝置清單，選取已連線 iPhone。 若清單空白，先確認手機解鎖、USB 連線與信任此電腦。
飛人模式 輸入座標或選擇已存地標，送出後立即切換定位。 需要 RSD tunnel 成功啟動。若失敗，看 logs\tunneld.err.log。
種花/路線 切到路線模式，依序點選路徑點，設定速度後開始。 路線移動會連續送定位。手機斷線或 tunnel 失效時會停止。
地標管理 儲存常用座標，之後可快速飛到該位置。 地標資料存放在可攜版資料夾內，搬移資料夾時會一起帶走。

Log 與狀態

新版會在背景執行，不會預設顯示 Pikomin Tunnel 或 Pikomin App 視窗。若啟動或定位異常，先看 logs 資料夾。

檔案 用途
logs\backend.log API 請求紀錄，例如裝置、定位、路線請求。
logs\backend.err.log 後端錯誤與定位失敗原因。
logs\tunneld.log tunnel 標準輸出。
logs\tunneld.err.log tunnel 權限、裝置連線與 RSD 錯誤。
logs\pids.json 背景程序 PID。stop.bat 會用它停止服務。

常見問題

瀏覽器打不開 localhost

先等 5 秒重新整理。仍失敗時查看 logs\backend.err.log，確認 app server 是否啟動。

定位顯示 RSD tunnel not available

代表 tunnel 沒成功。重新執行 run.bat 並允許系統管理員權限，確認手機解鎖。

Windows 防火牆提示

允許本機 server 即可。Pikomin 使用 localhost:5679 提供本機網頁與 API。

執行後想關掉

不要只關瀏覽器。請雙擊 stop.bat，它會停止背景中的 tunnel 和 app server。

交付檢查

  1. 1 確認壓縮包內有 run.bat、stop.bat、backend、frontend、python。
  2. 2 第一次啟動時允許系統管理員權限。
  3. 3 開啟頁面後確認裝置列表可讀取。
  4. 4 完成測試後執行 stop.bat，確認背景服務已停止。
